Revision Hub

Quick refreshers, mnemonics, flashcards, and pitfalls to watch. Review before your next practice session or real interview.

🔗 Big-O Essentials

  • O(1): Hash table lookup, array index access
  • O(log n): Binary search
  • O(n): Single loop, two pointers
  • O(n log n): Quick/Merge sort, Heapsort
  • O(n²): Nested loops, Bubble/Insertion sort

Tip: Always state your intuition before optimizing—interviewers value reasoning.

🎯 Array/String Patterns

  • Two Pointers: Valid Palindrome, Container With Most Water
  • Sliding Window: Longest Substring Without Repeat, Max Sum Subarray
  • Fast/Slow Pointers: Linked List Cycle, Happy Number
  • Merge Intervals: Meeting Rooms, Insert Interval

Pro tip: Visualize with pen/paper—don't code blind.

⭐ Behavioral STAR

Situation → Task → Action → Result

  • Failure: "What went wrong? What was your specific role? Precise actions taken? Quantified impact?"
  • Conflict: "I disagreed with X because Y. Proposed Z. Collaborated on solution."
  • Leadership: "Led N-person team. Scoped problem. Delegated tasks. Delivered 20% ahead."

Practice 2-3 stories per category. Time under 2min.

💬 Communication Reminders

  • Structure: "Three reasons: First... Second... Third."
  • Think Aloud: "Approaching with hashmap because time-critical lookup."
  • Edge Cases: Always mention: empty, single, max-size, all-duplicates.
  • Close Strong: "Time complexity: O(n), space: O(1). Alternative: pointers for O(n) time."

Avoid: filler words ("like", "um"), rambling without structure.

⚡ Quick Flashcards

Q: SQL indexes?
AnswerSpeed up SELECT WHERE/GROUP BY. Slow INSERT/UPDATE. Composite for multi-col queries.
Q: RESTful API status codes?
Answer200 OK, 201 Created, 400 Bad Req, 401 Unauthorized, 404 Not Found, 500 Internal.
Q: CAP Theorem?
AnswerConsistency, Availability, Partition tolerance. Pick 2.
